# 常规流程

按照[jetson-gpio](https://github.com/NVIDIA/jetson-gpio)的README即可简单完成

* 安装Jetson.GPIO库

  ```
  sudo pip install Jetson.GPIO
  ```

* 设置用户权限

  ```
  sudo groupadd -f -r gpio
  sudo usermod -a -G gpio {your_user_name}
  ```

* 运行示例程序[jetson-gpio/samples](https://github.com/NVIDIA/jetson-gpio/tree/master/samples)，如

  ```
  import Jetson.GPIO as GPIO
  import time
  
  # Pin Definitions
  output_pin = 18  # BCM pin 18, BOARD pin 12
  
  def main():
      # Pin Setup:
      GPIO.setmode(GPIO.BCM)  # BCM pin-numbering scheme from Raspberry Pi
      # set pin as an output pin with optional initial state of HIGH
      GPIO.setup(output_pin, GPIO.OUT, initial=GPIO.HIGH)
  
      print("Starting demo now! Press CTRL+C to exit")
      curr_value = GPIO.HIGH
      try:
          while True:
              time.sleep(1)
              # Toggle the output every second
              print("Outputting {} to pin {}".format(curr_value, output_pin))
              GPIO.output(output_pin, curr_value)
              curr_value ^= GPIO.HIGH
      finally:
          GPIO.cleanup()
  
  if __name__ == '__main__':
      main()
  ```

> Jetson GPIO库提供了四种给I / O引脚编号的方法（GPIO.setmode）
>
> - **BOARD**：物理引脚编号（40针接口顺序）
> - **BCM**：Broadcom SoC 的 GPIO 编号
> - **CVM**：CVM/CVB 连接器的信号名称
> - **TEGRA_SOC**：Tegra SoC 信号名称
>
> 查询Jeston型号
>
> ```
> cat /proc/device-tree/model
> ```
>
> ```
> NVIDIA Jetson Orin NX Engineering Reference Developer Kit Super
> ```
>
> 查看Jetpack版本
>
> ```
> jtop
> ```
>
> 右上角显示版本`Jetpack 6.2 L4T 36.4.3`

一般到这里你就可以根据[引脚定义图](https://jetsonhacks.com/nvidia-jetson-xavier-nx-gpio-header-pinout/)愉快地开始控制了，但如果你的Jetpack版本大于等于6.2的话，你会发现无论如何也无法切换到输出模式输出高电平或者低电平

#  6.0>=Jetpack版本<6.2.2

由于[下游内核驱动的变动](https://github.com/NVIDIA/jetson-gpio/issues/114#issuecomment-2353766759)，现在上层软件不能直接切换GPIO模式，所以要通过[busybox更改引脚复用器](https://github.com/NVIDIA/jetson-gpio/issues/114#issuecomment-2353766759)，在[Jetson Orin NX引脚表格](https://developer.download.nvidia.com/assets/embedded/secure/jetson/orin_nx/docs/Jetson_Orin_NX_and_Orin_Nano_series_Pinmux_Config_Template_v1.2.xlsm?t=eyJscyI6ImdzZW8iLCJsc2QiOiJodHRwczovL3d3dy5nb29nbGUuY29tLyJ9&__token__=exp=1782391281~hmac=0cf73ab1a982447189861715f4b8da3478c0a197e9229d75ddf70d62fd9455c6)中可以看到很多GPIO的默认模式是Input，所以需要进行修改

* 更新`Jetson.GPIO`

  ```
  pip install -U Jetson.GPIO
  ```

  > 2.1.7版本没有jetson-gpio-pinmux-lookup工具所以要更新，我更新到2.1.12版本就有了

* 使用[Pinmux 查找工具](https://developer.download.nvidia.com/assets/embedded/secure/jetson/orin_nano/docs/Jetson-Orin-Nano-DevKit-Carrier-Board-Specification_SP-11324-001_v1.3.pdf?t=eyJscyI6ImdzZW8iLCJsc2QiOiJodHRwczovL3d3dy5nb29nbGUuY29tLyJ9&__token__=exp=1782397432~hmac=1a43295d32e3bf26ba62e52eadae2cbf72b98bbf78daec5928c7eb16b2f1c2d4)可以快速查询到寄存器地址，如[简单直观的引脚定义图](https://jetsonhacks.com/nvidia-jetson-xavier-nx-gpio-header-pinout/)或下图所示

  ![image-20260625215831938](https://files.seeusercontent.com/2026/06/25/wX0e/image-20260625215831938.png)

  比如我要让GPIO09也就是7号针脚改为输出控制，

  ```
  $ jetson-gpio-pinmux-lookup 7
  WARNING: Carrier board is not from a Jetson Developer Kit.
  WARNNIG: Jetson.GPIO library has not been verified with this carrier board,
  WARNING: and in fact is unlikely to work correctly.
  WARNING: Carrier board is not from a Jetson Developer Kit.
  WARNNIG: Jetson.GPIO library has not been verified with this carrier board,
  WARNING: and in fact is unlikely to work correctly.
  GPIO Pin 7: Mux Register Address = 0x2448030
  ```

* 查询到地址为`0x2448030`，使用`devmem`工具查看寄存器值

  ```
  # 一台Jetson
  $ sudo busybox devmem 0x02448030
  0x0000005A
  # 另一台Jetson
  $ sudo busybox devmem 0x02448030
  0x0000045A
  ```

* 使用`devmem`工具设置寄存器

  ```
  # gpio09 板载7号
  sudo busybox devmem 0x02448030 w 0x004
  ```

  > 比如还有GPIO11也就是31号针脚
  >
  > ```
  > # 查询寄存器地址
  > $ jetson-gpio-pinmux-lookup 31
  > WARNING: Carrier board is not from a Jetson Developer Kit.
  > WARNNIG: Jetson.GPIO library has not been verified with this carrier board,
  > WARNING: and in fact is unlikely to work correctly.
  > WARNING: Carrier board is not from a Jetson Developer Kit.
  > WARNNIG: Jetson.GPIO library has not been verified with this carrier board,
  > WARNING: and in fact is unlikely to work correctly.
  > GPIO Pin 31: Mux Register Address = 0x2430070
  > # 查询寄存器的值
  > $ sudo busybox devmem 0x02430070
  > 0x00000058
  > # 修改寄存器的值 goio11 板载31号
  > $ sudo busybox devmem 0x02430070 w 0x004
  > ```

这么一番折腾之后，你会发现终于可以输出高电平与低电平了，

如果你想知道为什么是0x004，你可以在[这份文档](https://developer.nvidia.com/downloads/orin-series-soc-technical-reference-manual/)里看到，0x004也就是100，bit10是0也就是GPIO，bit6是0也就是禁用INPUT也就是使用OUTPUT

> 1. Set GPIO: Bit 10 = 0.
> 2. For the output, set Bit 4 = 0 ; Bit 6 = 0.
> 3. For Input, set Bit 4 = 1 ; Bit 6 = 1.

# Jetpack版本>=6.2.2

更新到 Jetpack 6.2.2 版本有所帮助，因为现在 jetson-io.py 提供了单独启用引脚 GPIO 的选项

```
sudo /opt/nvidia/jetson-io/jetson-io.py
```

![image-20260625221803690](https://files.seeusercontent.com/2026/06/25/qpB3/image-20260625221803690.png)

# PWM控制

```
sudo /opt/nvidia/jetson-io/jetson-io.py
```

`Configure Jetson 40pin Header`->`Configure header pins manually`

勾选你想要控制的pwm口，只有给定的几个可以用，比如`勾选pwm5（板载33号） pwm7（板载32） `
