hexo基本指令介绍

```bash
# 清除网页
hexo cl
# 生成网页
hexo g
# 网页本地发布
hexo s
# 网页提交到GitHub
heox d
```

# 使用ParticleX主题

> 主要是ParticleX加载很花时间（这可能就是因为好看而付出的代价吧），不太适合经常访问

仓库地址：[Hexo-Theme-ParticleX](https://github.com/argvchs/hexo-theme-particlex)

* 1.ParticleX主题的基本搭建
  
  按照仓库的readme配置好基本网页（主题安装+关闭 Hexo 自带 highlight）
  
  然后在`_config.yml`里面找到`theme: `改为`theme: ParticleX`
  
  > 注意：yml文件的冒号后面都有一个空格！！！！
  
  依次运行
  
  ```bash
  hexo cl
  hexo g
  hexo s
  ```

* 2.ParticleX主题的基本使用
  
  生成categories、about、tags
  
  ```bash
  hexo new page categories
  hexo new page about
  hexo new page tags
  hexo new post first
  ```
  
  然后就可以在source文件中发现如下文件结构
  
  ```text
  |---source
      |---_posts        #每一篇帖子的内容
      |---about        #about点开之后的内容
      |---categories
      |---tags
  ```
  
  categories和tags不用管，会自动生成
  
  about和_posts都是md文件格式
  
  `hexo new post <post-name>`会在_posts中再生成一个md文件（`<post-name>`为文件名
  
  post文件开头示例：
  
  ```md
  title: first
  date: 2022-09-27 19:52:59
  tags: 
    - 测试
  categories:
    - 踩坑
  description: "界面终于正常啦"
  ```
  
  背景、图标、评论配置参阅原仓库readme中：[配置文件说明](https://github.com/argvchs/hexo-theme-particlex)
  
  > tips：图标有两个元素，一个是name，一个是theme，在图标官网可以查阅
  > 
  > ​          比如GitHub图标的theme是brands

* 3.ParticleX主题的进阶配置
  
  原作者已经有了教程我也不过多赘述：https://argvchs.github.io/2022/04/17/hexo-blog-3/

# Q&A

* Permission denied

  ```
  hexo s
  INFO  Validating config
  FATAL Permission denied. You can't use port 4000.
  ```

  本以为是其他进程占用了4000端口，所以只要换个端口就行了`hexo s -p 5000`，然后就能用了

  不过后来查了查，[Hexo使用localhost:4000不能进行本地预览](https://www.jianshu.com/p/d3963da3946f)，发现4000端口没人用

  最后还得是国外论坛[net stop winnat](https://stackoverflow.com/questions/9164915/node-js-eacces-error-when-listening-on-most-ports)只要以管理员身份运行``net stop winnat``，``net start winnat``即可
